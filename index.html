pwc

Pentest Workspace Copilot

A tmux-native, human-in-the-loop workflow accelerator for authorized penetration testing on Kali Linux. It bootstraps structured engagement workspaces, turns intent into reviewed commands, captures evidence, remembers per-target state, and keeps a local audit trail — without ever auto-running anything.

⚠ For authorized systems, labs, and defensive/security testing only.

Features

tmux workbench

Per-engagement sessions with notes, shell, scans, web, and logs windows.

Command copilot

ask, run, fix, next, explain, review — natural language to reviewed commands.

Safety gate

Local risk engine and policy controls. Nothing runs without explicit approval.

Secret redaction

Keys, tokens, and credentials are stripped before any model call. Offline mode too.

Evidence capture

Outputs saved into structured folders with per-target knowledge tracking.

Local-first

Workspaces, knowledge, and an append-only audit log stay on your device.

Quick start

# install (pipx recommended; falls back to a venv)
$ ./install.sh

# wire up the shell hook, then open a new shell
$ echo 'source "$HOME/.local/share/pwc/shell/pwc.bash"' >> ~/.bashrc

# bootstrap an engagement and start the workbench
$ pwc init my-engagement
$ pwc target add 10.10.10.10
$ pwc start

# verify everything
$ pwc doctor

Commands

  • pwc init Bootstrap an engagement workspace
  • pwc target add Add and select a target
  • pwc start Create or attach the tmux workbench
  • pwc ask "..." Natural language to a proposed command
  • pwc run "..." Draft a command and offer to run it (gated)
  • pwc fix Propose a fix for the failed previous command
  • pwc next Suggest the next useful step
  • pwc explain Explain a command in plain English
  • pwc review Risk-review a command before running
  • pwc finding new Draft a finding from captured evidence
  • pwc history Show the local audit history
